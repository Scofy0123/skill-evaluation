# backend-estimation Skill 评估报告（2026-05-25）

## 执行摘要

> 本节先给结论：backend-estimation 是否可运行、是否有价值、主要风险在哪里。

本轮评测对象是 `backend-estimation`，目标是判断它能否基于 PRD 生成可审计的服务端估时结果，并评估它相比无 skill 临场估时是否有稳定收益。本轮使用当前在线 Sheet 第一列超链接抓取 PRD，主指标对齐在线「服务端预估」列；执行中未写入生产总览表，也未在真实 PRD 子节点下创建估时文档。

结论：backend-estimation 的运行框架、安全阻断、模板化详情产物和数据落盘能力较完整，适合继续作为服务端估时工作流沉淀。本轮 3 条在线服务端样本的 AI 规则回放与在线服务端预估整体接近，平均倍率为 1.0743x，平均绝对误差为 2.5333h。存在必要性目前是间接证据：它相对弱隔离无 skill 对照更可追溯、更容易复盘，但还缺少独立受控的无 skill 基线。

## Skill 优化方向

> 本节把评测结果反哺回 skill。每条优化方向都能追溯到数据或待补样缺口。

| 优化项 | 来源证据 | 建议修改 | 验证方式 |
| --- | --- | --- | --- |
| 增加独立无 skill 基线 | 存在必要性仍是间接证据 | 使用空白上下文、同样 PRD 输入运行无 skill 估时 | 对比 MAE、结构完整性、风险识别和人工认可 |
| 扩充信息不足样本 | 当前样本偏在线服务端成熟 PRD | 补充端侧不清、接口不明、跨端边界模糊的 PRD | 每个核心场景内部拆优化集 / 保留集 |
| 明确人工认可档位 | 人工评测仍待填写 | 让服务端负责人按“可直接用 / 需小改 / 需大改 / 不可用”填写 | 更新指标结果记录并复测分歧样本 |
| 修改 skill 后补样复测 | 估时规则后续会调整 | 每次修改估时规则、模板或边界判断时，补充能击中新改动点的样本 | 重跑受影响优化集，同时抽测保留集，确认优化点确实改善且未破坏原能力 |
