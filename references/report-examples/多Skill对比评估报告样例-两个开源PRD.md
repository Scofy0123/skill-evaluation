# 两个开源 PRD Skill 评估报告

## 执行摘要

> 本节先给结论：当前证据下哪个候选更好用、适合什么场景、差距在哪里、是否优于无 skill 基线。

本轮对比对象是两个开源 PRD skill：`github/awesome-copilot/skills/prd` 与 `phuryn/pm-skills/pm-execution/skills/create-prd`。评测目标是判断它们在 PRD 生成任务中的可唤起性、PRD 主功能产物质量、运行稳定性、trace 可解释性，以及相比无 skill 基线是否有稳定收益。

阶段性结论：在已真实完成的模糊需求样本中，两个 skill 都能产出可评审 PRD；无 skill 基线同样很强，因此两个 skill 的存在必要性尚未被当前样本证明。可唤起性方面，awesome-copilot/prd 静态判断为 24/24，phuryn/create-prd 为 23/24，phuryn 因 description 覆盖 “review existing PRD” 语义，在相似干扰样本中边界略宽。PRD 产物风格上，awesome 更偏工程化和风险约束，phuryn 更偏产品叙事和业务表达；当前证据不足以给出全场景最终排名。

| 层级 | 检查项 | awesome-copilot/prd | phuryn/create-prd | 证据 |
| --- | --- | --- | --- | --- |
| 运行框架层 | 可唤起性 / 路由选择 | ✅ 24/24 | 部分满足 23/24 | `可唤起性静态仿真结果.csv` |
| 运行框架层 | 独立运行成功产物 | 8 条 | 7 条 | `评测运行状态.json` |
| 运行框架层 | 事件日志与错误可观测 | ✅ | ✅ | `codex-jsonl-logs/`、`运行错误日志/` |
| 能力链路层 | PRD 结构完整性 | ✅ | ✅ | 已完成模糊需求样本的规则评判与模型辅助复核 |
| 能力链路层 | 输出保存行为 | ✅ | ✅ | PRD 产物落盘到 `生成PRD产物/` |
| 业务效果层 | 模糊需求 PRD 质量 | ✅ | ✅ | 6 个三方成对比较样本中两者基本可用 |
| 业务效果层 | 相比无 skill 基线 | 未证明 | 未证明 | 6 个三方成对比较样本中 skill 未高于 baseline |
| 业务效果层 | 人工业务认可 | 人工待评 | 人工待评 | `人工评测填写表.csv` |

| 问题 | 当前结论 | 证据 | 限制 |
| --- | --- | --- | --- |

| 适合场景 | awesome 更适合工程化 PRD、风险和技术边界；phuryn 更适合早期产品叙事 | 样本产物风格、模板要求 | 需在半结构化、详细需求、变更约束场景补跑 |

| 是否优于无 skill 基线 | 当前未证明 | 6 个成对样本 skill 分数未高于 baseline | 基线也由强模型生成，需人工复核 |

## 评测目标与指标体系

| 指标 | 所属层级 | 计算规则 | 评判方式 | 评判规则 | 大语言模型及版本 | 评分细则路径或说明 | 本轮结果 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 可唤起性 | 运行框架层 | 符合预期的路由判断数 / 路由样本数 | 规则评判 | 读取 SKILL description，对直说、隐含、相似干扰、负例样本做静态判断 | 不涉及模型评判 | 正例应触发，负例和相似干扰不应误触发 | awesome 24/24，phuryn 23/24 |
| 独立运行成功率 | 运行框架层 | 成功产物数 / 已启动运行数 | 规则评判 | `codex exec --json` 退出码 0 且输出文件通过基础内容检查 | Codex CLI 默认模型，未输出精确模型名 | 产物大于 600 字且含 PRD/需求/目标/验收等关键内容 | 21 条成功产物，4 条失败，3 条中断/未完成 |

| 存在必要性 | 业务效果层 | skill 输出是否稳定优于无 skill 基线 | 规则评判、模型评判、人工评判 | 同输入独立运行 baseline；比较结构、可执行性、风险和可复现性 | baseline 与候选使用同一 Codex CLI 默认模型 | 只有独立基线成对样本计入 | 当前未证明两个 skill 优于 baseline |

## Skill 优化方向

> 本节把评测结果反哺回 skill 和评测资产。

| 优化项 | 来源证据 | 建议修改 | 验证方式 |
| --- | --- | --- | --- |
| 收窄 phuryn 可唤起边界 | ROUTE-SIM-03 误触发 | description 中区分“创建 PRD”和“评审已有 PRD”职责 | 重跑相似干扰型路由样本 |
| 为两者补充“为什么优于 baseline”的强约束 | 6 个成对样本未高于 baseline | 让 skill 输出更明确的澄清问题、假设、风险、验收和可追溯结构 | 同样本重跑三方矩阵，比较人工认可率 |
| 补齐场景体系 | 当前正式质量结论只覆盖模糊需求 | 按需求输入成熟度补充半结构化、详细需求、变更约束样本 | 每个核心场景内拆优化集 / 保留集后重跑 |
| 修改后补样复测 | 后续可能调整 skill description 或 PRD 模板 | 针对被修改点新增样本，例如误触发样本、路线图缺失样本、AI 风险样本 | 重跑受影响优化集，同时抽测保留集，确认优化符合预期且不破坏原能力 |
