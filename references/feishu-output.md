# 飞书输出规范

当用户要求创建或更新飞书文档时，使用这份说明。

## 写作规则

- 新评估报告优先新建文档。
- 更新已有报告时，优先做章节级插入/替换，不整篇覆盖，以便保留评论。
- 除非用户要求，不在正文里提内部来源文档名称；正文贯彻方法，不展示方法来源清单。
- 每个主要章节开头使用引用块，帮助不懂评测的读者理解本节作用。
- 指标、样本、步骤、优化项优先用表格。
- 如果表格用于后续人工复核，把填写提示放在表头，单元格留空。
- 最终飞书正文必须是面向读者的评估结论和证据，不要把本 skill 的操作规范、写作口径或方法论提示原样放进去。比如 Codex 事件日志规则应写成“本轮已采集 session JSONL，路径为...；真实写入链路未执行，因此该链路未评估”，不要写成“如果使用 Codex 跑 skill，应优先采集...”或“报告后续只写...”。写入前搜索 `如果使用`、`报告后续`、`应优先`、`只写`、`不要`、`必须先` 等词，确认它们不是内部指导语。

## 飞书编辑流程

1. 编辑前先拉取当前文档。
2. 保存本地副本。
3. 识别标题、图片 token、表格位置。
4. 准备小块 Markdown 片段。
5. 做局部章节更新。
6. 再次拉取，检查标题、重复章节和关键新增术语。
7. 再次拉取后搜索指导性元话语，确认最终文档没有把内部评测规范误当作报告正文。

除非文档没有重要评论，否则避免整篇替换。

## 评测文件上传

- 评测结果文件默认逐个上传到飞书云盘，不要把多个结果文件打成 zip 后只上传压缩包。
- 只有用户明确要求“打包”“压缩包”“zip”时，才额外上传压缩包；即使上传压缩包，也应保留关键结果文件的单独上传版本。
- 面向用户展示或交付的飞书附件文件名必须使用中文名称。可以在本地保留 `metric_results.json`、`sample_results.csv` 等英文技术文件名，上传时映射为 `指标结果.json`、`样本结果.csv`、`真实样本.csv` 等中文名称。
- 创建飞书云盘文件夹后，上传完成要回读文件夹列表，确认每个单独文件都存在；如果误传了 zip 且用户不需要，应删除 zip 并重新上传单独文件。
